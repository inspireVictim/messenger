# Messenger.Infrastructure

Реализации всего, что «снаружи» домена: БД, шифрование, JWT.

## Твоя зона

- `AppDbContext`, маппинг сущностей, миграции.
- Реализации репозиториев: `UserRepository`, `ChatRepository`, `MessageRepository`.
- `AesEncryptionService`, `JwtService`.

Инфраструктурный слой зависит от Domain и Application (реализует интерфейсы из Domain, регистрируется для использования в Application).

## Правила

- Все интерфейсы из Domain реализуются здесь. Не добавлять новые интерфейсы в Domain без согласования.
- Шифрование и JWT здесь, потому что это детали реализации (ключи, алгоритмы, секреты), а не доменная логика.

## Куда не лезть

Сущности и интерфейсы — в Domain. Бизнес-логика — в Application. Маршруты и контроллеры — в WebAPI.
